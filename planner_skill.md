# Paperclip Company Planner Skill 设计草案

---

## 一、可行性分析

**完全可行**。这本质上是一个"公司规划师"skill，可以：

1. **输入**：人类的高层目标
2. **处理**：AI 进行架构推演、角色设计、预算规划
3. **输出**：完整的 Paperclip 公司配置 + Agent 配置 + 目标/任务分解

---

## 二、Skill 设计草案

### Skill 元信息

```yaml
---
name: paperclip-company-planner
description: |
  设计 Paperclip 公司组织架构。根据你的业务目标，自动规划：
  - 组织架构和 Agent 角色
  - 目标层级和任务分解
  - 预算分配
  - Agent 配置参数
  
  适用于：新公司创建、团队扩张规划、项目制团队搭建。
---
```

## 三、人类需要提供什么？

### 必需信息

| 信息 | 为什么需要 | 示例 |
|------|-----------|------|
| **业务目标** | 决定公司类型和核心任务 | "写一本 OpenClaw 教程书" |
| **时间周期** | 决定节奏和紧迫度 | "6 个月内完成" |
| **预算范围** | 决定团队规模和能力 | "$500-1000/月" |

### 推荐信息

| 信息 | 为什么需要 | 示例 |
|------|-----------|------|
| **团队规模偏好** | 精简 vs 完整团队 | "小团队，5人以内" |
| **技术栈/工具** | 决定 Agent adapter 类型 | "使用 OpenClaw" |
| **市场/用户** | 决定营销策略 | "面向开发者" |
| **质量标准** | 决定审核流程复杂度 | "代码必须可运行" |

### 可选信息

| 信息 | 为什么需要 | 示例 |
|------|-----------|------|
| **现有资源** | 复用已有配置 | "我有一个 OpenClaw 实例" |
| **风险偏好** | 决定预算控制策略 | "严格控制，不要超支" |
| **协作模式** | 决定审批流程 | "我想要最终审核权" |

### 详细思考清单

**1. 业务分析**
```
问题：
- 这是产品开发、内容创作、服务交付、还是研究项目？
- 核心交付物是什么？（代码、文档、设计、数据）
- 质量要求有多高？
- 是否需要外部协作？

决策：
- 公司类型标签
- 核心工作流模式
- 质量控制级别
```

**2. 组织架构推导**
```
问题：
- 需要哪些职能？（技术、内容、设计、营销、运营）
- 哪些职能可以合并？（如 CTO + 技术负责人）
- 哪些必须独立？（如需要独立审核的内容编辑）
- 管理层级应该多深？

决策：
- 角色列表
- 汇报关系树
- 管理幅度
```

**3. Agent 配置推理**
```
问题：
- 这个角色需要什么能力？（编程、写作、分析、设计）
- 应该用什么 adapter？（openclaw、claude-local、process）
- 心跳频率？（高频交互 vs 低频检查）
- 需要多少上下文？（thin vs fat）

决策：
- adapterType
- adapterConfig 参数
- contextMode
- 心跳间隔
```

**4. 目标层级设计**
```
问题：
- 最终目标是什么？
- 中间里程碑有哪些？
- 如何分解到团队和个人？
- 如何衡量进度？

决策：
- Goal 层级结构
- KPI 定义
- 里程碑时间点
```

**5. 任务依赖分析**
```
问题：
- 任务之间有什么依赖？
- 哪些可以并行？
- 哪些有审批/审核环节？
- 预期周期是多长？

决策：
- Issue 结构
- 前置依赖
- 状态流转
- 预估工期
```

**6. 预算优化**
```
问题：
- 核心任务由谁完成？
- 核心角色应该分配更多预算吗？
- 如何避免预算浪费？
- 预警策略？

决策：
- 预算分配比例
- 预警阈值
- 调整机制
```

## 五、Skill 流程设计

```markdown
# Company Planner Skill 流程

## Phase 1: 信息收集（交互式）

### 必问问题
1. 你的业务目标是什么？（一句话描述）
2. 预期完成时间？
3. 月度预算范围？

### 追问问题（根据回答）
- 如果是产品：技术栈是什么？目标用户是谁？
- 如果是内容：内容类型？质量标准？
- 如果是服务：服务对象？交付形式？

## Phase 2: 方案生成

1. 分析业务类型，匹配最佳组织模式
2. 生成组织架构图
3. 定义每个角色的职责和配置
4. 分解目标层级
5. 规划预算分配

## Phase 3: 方案确认

展示：
- 组织架构图
- Agent 配置摘要
- 目标分解
- 预算分配
- 预估总成本

询问：
- 是否调整角色数量？
- 是否调整预算分配？
- 是否需要额外角色？

## Phase 4: 生成可执行配置

输出：
1. 完整的初始化脚本
2. 每个 Agent 的配置 JSON
3. SOUL.md / HEARTBEAT.md 模板
4. 目标和任务创建命令
```

## 六、示例交互

```
Human: 我想创建一个公司来开发一个 SaaS 产品

Skill: 好的，让我了解更多细节：

1. 这个 SaaS 产品的核心功能是什么？
2. 目标用户是谁？
3. 预期上线时间？
4. 月度预算范围？

Human: 
1. 项目管理工具，面向小团队
2. 小团队、创业公司
3. 6 个月内 MVP 上线
4. $800/月
5. 兼职，主要做产品决策

Skill: 基于你的需求，我设计了以下组织架构：

【组织架构】
```
        CEO (你)
           │
     ┌─────┼─────┐
     │     │     │
    CTO   CPO   CMO
     │     │     │
   开发x2 设计x1 运营x1
```

【角色说明】
- CTO: 技术架构、代码审核、开发进度管理
- CPO: 产品设计、用户研究、功能规划
- CMO: 市场推广、用户获取、社区运营

【预算分配】（$800/月）
- CTO: $150
- 开发 x2: $200（各$100）
- CPO: $100
- 设计: $80
- CMO: $70
- 储备: $200

【目标分解】
- 公司目标: 6个月内发布 MVP
  - 技术目标: 完成核心功能开发
  - 产品目标: 完成用户验证
  - 营销目标: 获取 100 种子用户

是否需要调整？
- [ ] 调整团队规模
- [ ] 调整预算分配
- [ ] 添加/删除角色
- [ ] 生成完整配置
```

## 七、实现建议

### Skill 文件结构

```
skills/
└── company-planner/
    ├── SKILL.md              # 主 skill 文件
    ├── templates/
    │   ├── soul-ceo.md       # CEO SOUL.md 模板
    │   ├── soul-cto.md       # CTO SOUL.md 模板
    │   ├── heartbeat.md      # 心跳行为模板
    │   └── init-script.sh    # 初始化脚本模板
    ├── patterns/
    │   ├── content-company.json    # 内容创作公司模式
    │   ├── saas-company.json       # SaaS 公司模式
    │   ├── service-company.json    # 服务公司模式
    │   └── research-company.json   # 研究项目模式
    └── prompts/
        ├── gather-info.md     # 信息收集 prompt
        ├── analyze.md         # 分析 prompt
        └── generate.md        # 生成 prompt
```

### 核心逻辑

```typescript
// 伪代码
async function planCompany(userInput: UserInput): Promise<CompanyPlan> {
  // 1. 分析业务类型
  const businessType = analyzeBusinessType(userInput);
  
  // 2. 加载匹配模式
  const pattern = loadPattern(businessType);
  
  // 3. 根据预算调整规模
  const scaledPattern = scalePattern(pattern, userInput.budget);
  
  // 4. 生成具体配置
  const agents = generateAgents(scaledPattern, userInput);
  const goals = generateGoals(scaledPattern, userInput);
  const tasks = generateTasks(scaledPattern, userInput);
  
  // 5. 计算预算
  const budget = calculateBudget(agents, userInput.budget);
  
  return { agents, goals, tasks, budget };
}
```

## 九、总结

| 维度 | 内容 |
|------|------|
| **人类提供** | 业务目标、时间、预算、技术栈、质量要求 |
| **AI 思考** | 组织架构、角色定义、目标分解、任务拆分、预算优化 |
| **Skill 输出** | 完整的公司配置、Agent 配置、创建脚本 |

这个 skill 的核心价值是**将"组织设计"这个复杂决策过程变成结构化的对话**，让用户不需要自己思考架构，只需回答关键问题。
